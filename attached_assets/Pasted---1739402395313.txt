Задача: Разработка десктопного микросервисного приложения для контроля личной жизни

Цель: Создать простое, понятное и логичное приложение с использованием как минимум двух паттернов проектирования. Это приложение будет представлять собой домашнюю работу для отчета преподавателю.

Требования к приложению:

Паттерны проектирования:

Использовать как минимум два паттерна проектирования, которые будут полностью и обоснованно интегрированы в приложение.
Основные функции:

Личный кабинет: Пользовательский интерфейс для управления учетной записью.
Регистрация: Возможность создания нового аккаунта с валидацией данных.
База данных: Хранение информации о пользователях, задачах, привычках и книгах.
Домашняя страница: Генерация случайных идей дня с подбором статей из интернета.
Интеграция API: Возможность подключения API локальной нейросети (например, LM Studio или Ollama).
Личная библиотека: Хранение книг с отслеживанием прогресса чтения и возможностью делать заметки на нужных страницах. Возможно, потребуется реализовать PDF-читалку.
Трекер привычек: Функция с ежедневными напоминаниями и отметками о выполнении привычек.
Планировщик задач:
Реализация метода "бегущий список" для управления задачами.
Возможность быстрого добавления и изменения приоритетов задач.
Использование маркировки задач с помощью ключей (например, крестики для отмены, галочки для выполненных, точки для текущих и стрелочки для перенесенных задач).
Возможность цветной маркировки задач для удобства визуализации.
Дополнительный функционал:

Календарь: Интеграция календаря для планирования задач и привычек.
Статистика: Отображение статистики по выполнению задач и привычек.
Напоминания: Настройка уведомлений о предстоящих задачах и привычках.
Темы оформления: Возможность выбора различных тем оформления для пользовательского интерфейса.
Синхронизация: Возможность синхронизации данных между устройствами (если это возможно в рамках проекта).
Документация и комментарии:

Весь код должен быть тщательно прокомментирован, чтобы объяснить его логику и функциональность. Комментарии должны быть понятными и доступными для понимания, учитывая мой низкий уровень подготовки. Это поможет мне лучше разобраться в проекте и его структуре.
Заключение: Приложение должно быть интуитивно понятным и эффективным, обеспечивая пользователю удобный инструмент для контроля личной жизни и управления задачами.
